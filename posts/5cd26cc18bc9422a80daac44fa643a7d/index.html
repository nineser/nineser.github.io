层叠
继承(inheritance)是从一个元素向其后代元素传递属性值所采用的机制。确定应当向一个元素应用哪些值时，用户代理不仅要考虑继承，还需要考虑声明的特殊性，另外需要考虑声明本身的来源。这个过程就称为层叠(cascade)。
特殊性
选择器的特殊性由选择器本身的组件确定。特殊性表述为4个部分：0，0，0，0。 一般地，第一个0是为内联样式声明保留的，特殊性最高。 对于选择器：
各个 ID 属性值，加 0，1，0，0
各个类属性值、属性选择或伪类，加 0，0，1，0
对于各个元素和伪元素，加 0，0，0，1
通配选择器对特殊性没有贡献，有0特殊性
结合符和继承根本没有特殊性，连0都没有
重要性
有时某个声明可能非常重要，超过了所有其他声明，即重要声明。
p.dark{color: #333 !important;}
CSS
如果一个重要声明和一个非重要声明冲突（包括内联样式声明），胜出的总是重要声明。
继承
<li><a href=""></a></li>
HTML
li{color : white;}
CSS
上面的例子中的color:white可能不会在链接上显现出来，但原因不是 a 元素没有继承，而是因为浏览器的样式表可能含有a:link{color:blue}
权重
声明的权重由大到小的顺序依次为： 1. 读者的重要声明 2. 创作人员的重要声明 3. 创作人员的正常声明 4. 读者的正常声明 5. 用户代理的正常声明
层叠规则
1.
找出所有相关的规则
2.
!important > 非!important
读者的重要声明 > 创作人员的重要声明
创作人员的正常声明 > 读者的正常声明 > 用户代理的正常声明
3.
较高特殊性 > 较低特殊性
4.
后出现的 > 先出现的
导入样式为前，主样式为后
