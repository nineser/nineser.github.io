GraphQL
历史
2012 facebook 开发，提供 API 查询的语言，用在 facebook 平台上，
2015 年开源，2016 下半年宣布可以用于生产
2017 年，restful api 架构的典范，github 开放了使用 GraphQL 开发的第四代接口，使得 GraphQL 引起了业界的广泛关注。
背景
Facebook 开发 GraphQL 的最初原因是移动用户的增加、低功耗设备和松散的网络。GraphQL 最小化了需要网络传输的数据量，从而极大地改善了在这些条件下运行的应用程序。
特点
请求你所要的数据不多不少 获取多个资源只用一个请求
优点
优秀的性能体验（移动端）
Facebook 开发 GraphQL 的最初原因是移动用户的增加、低功耗设备和松散的网络。GraphQL 最小化了需要网络传输的数据量，从而极大地改善了在这些条件下运行的应用程序。
兼容各个平台
前端框架和平台运行客户端应用程序的异构环境使得我们在构建和维护一个符合所有需求的 API 变得困难，使用 GraphQL 每个客户机都可以精确地访问它需要的数据
满足产品可以快速迭代频繁更新的需求
对于 REST api，服务器公开数据的方式常常需要修改，以满足客户端的特定需求和设计更改，而且通常改动比较大，于是就需要对 API 版本进行迭代，这肯定是会影响产品快速开发和迭代的，特别是大型项目
总结
移动性能优秀 适合图状数据 并不会替代 restful
