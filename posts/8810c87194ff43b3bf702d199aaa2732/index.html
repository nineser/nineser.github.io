margin与尺寸
CSS margin 可以改变容器的尺寸，元素的尺寸又分为 可视尺寸和占据尺寸。
border 旁边的实线所包围的区域是 可视尺寸 clientWidth，
margin 旁边的虚线所包围的区域是 占据尺寸 outerWidth
margin 与可视尺寸
适用于没有设定宽高的普通 block 水平元素（非 float 等能让元素 block 化的属性）
只适用于水平方向尺寸
margin 与占据尺寸
block/inline-block 水平元素均适用
与有无设定宽高值无关
适用于水平方向和垂直方向
margin 与 百分比单位
普通元素的百分比 margin 的值都是相对于容器宽度计算。
绝对定位元素的百分比 margin 的值是相对于第一个定位祖先元素的宽度计算
需要注意的是，其偏移的起点是从父级开始的。
宽度 2：1 自适应矩形
.box {background: #eee;overflow: hidden;} .box > div {margin: 50%;}
CSS
margin 与重叠
空 block 元素 margin 重叠其它条件
元素没有 border
元素没有设置 padding
里面没有 inline 元素
没有 height/min-height
计算规律：正正取大，正负相加，负负最负

表单（写列表/垂直布局）设计时，可以给表单容器添加同时添加 margin-top 和 margin-bottom
margin: auto
auto，自动分配 在会自动填充的方向上 的剩余空间。
举个例子，默认情况下，块状元素的 width 等于父元素的 width。
是因为：width:auto 自动填充了水平方向上的空间。
在盒模型的 7 大水平的属性中，只有三个可以设置为 auto：width、margin-left 或 margin-right。
block 元素的 7 大水平的属性值之和 = 父元素的 width。
只要满足此等式，就算子元素左右外边距为负值都是可以的。
由上面的等式可知：
如果有一个 block 元素，水平属性设置了 width、margin-left、margin-right：
如果这三个值都为 auto 时，那么它的两个外边距都会设置为 0，使它的 width 尽可能宽（自动填充父元素的 width）。
如果这三个值都为非 auto 的某个值，此时总会设置 margin-right:auto，由这个 auto 来自动分配剩余的空间，使元素的占据的宽度等于其父级的 width。
如果只有 width 设置为固定值（不超过父元素的 width 值），其余两个设置为 auto，那么它就不会自动填充父元素的 width 了，于是设置为 auto 的属性会自动分配多余的空间。从而使元素所占据的尺寸等于父元素的 width。
由上面的内容可以知道，左右外边距 auto 可以让块元素水平居中，但是，垂直方向上的上下外边距设置为 auto 是不可以的让块状元素垂直居中的。 因为 auto 自动分配剩余空间，块状元素在垂直方向上默认是不会自动填充父元素的 height 的。
因此要想让元素垂直水平居中，就要让元素在四个方向上都自动填充。
于是，利用绝对定位可以做到。
原理是：
通过 absolute 化子元素，设置上下左右的属性值为 0；
让其具有自动填充性，然后设置宽度，就有了剩余空间，
接着再使用 margin:auto, 分配剩余空间。
/*（IE8+）*/ .father { height: 200px; position: relative; } .son { position: absolute; top: 0;right: 0;bottom: 0;left: 0; width: 500px;height: 100px; margin: auto; }
CSS
margin 负值
对于 static 元素（默认设置了 width）
margin-top / margin-left 为负
元素将向指定的方向移动。
margin-right / margin -bottom 为负
此元素本身不移动，后续元素向 左/上 移动，覆盖此元素。
并不会减少元素渲染所需要的高度，JS 获取的高度也是渲染元素所需要的高度
减少的是渲染时自身给 css 读取的高度，因此后续的元素会往上移动
没有设置 width（width: auto）
margin-left/margin-right 的负值 会增加元素的宽度（表现是 向左和向右 拉伸元素）。
对于 float 元素间，margin 负值可以让它们之间产生重叠。 例如：
两列 float 元素，一列定宽，一列 100% 宽；可想而知，他们是不可能共处一行。
但利用 margin 负值，可以使 100% 宽的一列覆盖定宽的一列，从而使他们能够共处一行。
负值的应用
列表两端对齐
<div class="box"> <div class="ul"> <div class="li"></div> <div class="li"></div> <div class="li"></div> </div> </div>
HTML
.box { width: 1200px; margin: auto; background: skyblue; } .ul { overflow: hidden; /*去除浮动影响*/ } .li { width: 380px; height: 300px; margin-right: 20px; float: left; background: #602aa5; }
CSS
此时的结果：
可以发现右侧多出 20 像素宽的蓝色区域，要想去掉它:
如果仅仅只是将多余的 20 像素平分到每个列表项上的话，第三个列表项就会下移到第二行。
因此，我们需要增加父级 ul 的宽度，但不是直接设置宽度，而是利用负值的 margin，
即 margin-right:-20px;
.box { width: 1200px; margin: auto; background: blue; } .ul { /*去除浮动影响*/ overflow: hidden; margin-right: -20px; } .li { width: 386.666666px; height: 300px; margin-right: 20px; float: left; background: green; }
CSS
两栏等高布局
思路如下：
1.
两列 float ，定不定 width 看需要；
2.
根据需要选取 padding-bottom 值，给两列撑开一定高度；
3.
利用 margin-bottom 负值，抵消撑开的高度，让后面的元素回到原来的位置；
4.
此时给元素染上背景色，就可以看出，严格来说，并不是 padding-bottom 值被抵消了，而是 margin-bottom 负值减少了其自身供 CSS 渲染页面时读取的高度，让后续元素跟了上来。
5.
元素的实际高度应该是加上 padding-bottom 后的高度，但上面说了，margin-bottom 的负值减少了供 CSS 读取的高度，所以，父元素能以最高的高度通过 overflow 隐藏掉多余的部分
<div class="box"> <div class="child-orange"> <div class="li">左黄</div> <div class="li">左黄</div> <div class="li">左黄</div> </div> <div class="child-green"> <div class="li">右绿</div> <div class="li">右绿</div> <div class="li">右绿</div> <div class="li">右绿</div> </div> </div>
