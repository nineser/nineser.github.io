类组件无法使用查询 sql hock
有时我们的类组件需要 state 进行状态管理，但类组件没法使用Gatsby 的查询 sql hock
解决方式如下
使用上层组件查询，将结果传递给需要子组件。
子组件使用 state 进行状态管理。

sharp 包安装错误
运行 gatsby develop，报错，提示删除包目录下的 sharp 包，接着运行 npm install，观察错误。
按提示 npm install 后，错误关键句如下
PS D:\\Workspace\\blog> npm install > sharp@0.23.4 install D:\\Workspace\\blog\\node_modules\\sharp > (node install/libvips && node install/dll-copy && prebuild-install) || (node-gyp rebuild && node install/dll-copy) info sharp Using cached D:\\devSoftware\\node-v10.16.3-win-x64\\node_cache\\_libvips\\libvips-8.8.1-win32-x64.tar.gz ERR! sharp TAR_BAD_ARCHIVE: Unrecognized archive format info sharp Attempting to build from source via node-gyp but this may fail due to the above error info sharp Please see <https://sharp.pixelplumbing.com/page/install> for required dependencies
Shell
解决方式：
删除对应目录下的 libvips 文件，运行 npm install
PS D:\\Workspace\\blog> npm install > sharp@0.23.4 install D:\\Workspace\\blog\\node_modules\\sharp > (node install/libvips && node install/dll-copy && prebuild-install) || (node-gyp rebuild && node install/dll-copy) info sharp Downloading <https://github.com/lovell/sharp-libvips/releases/download/v8.8.1/libvips-8.8.1-win32-x64.tar.gz>
PowerShell
从上述命令行中复制对应 sharp-libvips 地址，手动下载，然后放入 node 缓存目录的_libvip 文件中
PS D:\\Workspace\\blog> npm install > sharp@0.23.4 install D:\\Workspace\\blog\\node_modules\\sharp > (node install/libvips && node install/dll-copy && prebuild-install) || (node-gyp rebuild && node install/dll-copy) info sharp Using cached D:\\devSoftware\\node-v10.16.3-win-x64\\node_cache\\_libvips\\libvips-8.8.1-win32-x64.tar.gz info sharp Creating D:\\Workspace\\blog\\node_modules\\sharp\\build\\Release info sharp Copying DLLs from D:\\Workspace\\blog\\node_modules\\sharp\\vendor\\lib to D:\\Workspace\\blog\\node_modules\\sharp\\build\\Release prebuild-install WARN install connect ETIMEDOUT 52.216.98.251:443
PowerShell
prebuild install 错误，那就手动全局下载 prebuild-install 包
npm install prebuild-install -g
Bash
最后，运行 npm install，没有错误了。
