设备像素
设备像素又称物理像素（physical pixel）,设备能控制显示的最小单位
设备独立像素(density-independent pixel)
也称密度无关像素，可以认为是计算机坐标系统中的一个点
这个点代表可以 由程序使用并控制 的虚拟像素，然后能够被相关系统转换为物理像素。
比如：CSS像素 => 中间系统 => 物理像素
设备像素比(device pixel ratio)
CSS 中的像素是一个相对值，不是绝对值，因此 1px 的 CSS 像素并不一定就等于 1px 的物理像素。
CSS 中的像素单位是抽象的，最终的显示取决于物理设备（中间系统）。
物理设备根据其自身的某种规则，决定该采用几个物理像素去显示 1px 的 CSS 像素
而这个规则其实也就是设备像素比 (简称dpr) 。
在 水平/垂直 方向上
设备像素比 = 物理像素 / 设备独立像素
在dpr为 2 的设备下，1 个 css 像素，设备会使用 2 个物理像素去呈现它的宽和高，最终的结果也就是一个CSS像素，设备使用了4个物理像素去呈现。
位图像素
一个位图像素是栅格图像 (如：png,jpg,gif等) 最小的数据单元，每个位图像素都包含着一些自身的显示信息(显示位置，颜色值，透明度等)。
理论上，1个位像素对应1个物理像素，这是图片最完美的展示。这在dpr为1的屏幕设备下，没有问题；但在retina屏幕下，就会出现位图像素点不够，导致图片模糊的情况了，类似将小图片拉扯大，变模糊的效果。
由于单个位图像素不可再进行分割，所以设备只能就近取色，从而导致图片变得模糊。
所以，对于图片高清问题，通常的方案就是使用多倍图片。如在dpr为2的设备下，即对于200_300(css pixel)的图片，就需要提供 400\_600 的图片(@2x)。
💡
如果在普通屏幕下，对于200_300(css pixel)的图片，所对应的物理像素个数也就是200_300，假设使用2倍图片，其位图像素个数则是200*300*4；也就是会出现一个物理像素点对应4个位图像素点，在这个时候，设备对它的取色会通过一定的算法（显示结果就是一张只有原图像素总数4分之1）进行downsampling，肉眼看上去图片虽不会模糊，可以接受，但有时也会觉得图片缺少一些锐利度，或者是色差。
