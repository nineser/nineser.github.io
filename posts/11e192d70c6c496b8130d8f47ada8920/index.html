默认的文档流是从上往下，表现为单列。因此两栏布局，必定是要从影响文档流入手。
也就是说，要实现两栏布局，就需要使用浮动以及定位或者本身自带列的表格。
左列定宽，右列自适应
float + margin(+)
用 float 和 width 限制左行(列)的宽度，右行(列)上移，与左列共处一行；
在利用 margin 正值减少右行(列)的宽度。
其实，如果右列高度永远比左列低，右列并不需要 margin 值，即单纯的文字环绕效果。
这里不一定要用 margin 值，还可以给右行(列)添加 overflow，使其成为 BFC，以来消除浮动的影响
/*IE6会有3px的bug*/.left { float: left; width: 200px; } .right { margin-left: 200px; }
CSS
float + margin(-)
利用 float 和 width 实现左行(列)，右行(列)上移，与左行(列)共处一行； 给右行(列)的盒子添加 float + width，由于 width 值过大，右行(列)的盒子下移； 利用 margin 负值使右行(列)的盒子向左移动，能够覆盖左行(列)的宽度，因此其上移； 最后将利用 margin 正值减少右列(列)宽度，使覆盖在其下面的左行(列)显示出来
<html> <head></head> <body> <div class="parent"> <div class="left"> left </div> <div class="right-box"> <div class="right"> right </div> </div> </div> </body> </html>
HTML
.left { float: left; width: 500px; background: red; } .right-box { float: right; width: 100%; margin-left: -500px; } .right { margin-left: 500px; background: blue; }
CSS
table
.parent { display: table; table-layout: fixed; width: 100%; /*display:table-cell下连续单词字符换行问题修复*/ word-wrap: break-word; } .left { width: 100px; } .right,.left { display: table-cell; }

等高布局
1.
两列 float ，定不定 width 看需要；
2.
根据需要选取 padding-bottom 值，给两列撑开一定高度；
3.
利用 margin-bottom 负值，抵消撑开的高度，让后续元素回到原来的位置；
4.
此时给元素染上背景色，就可以看出，严格来说，并不是 padding-bottom 值被抵消了，而是 margin-bottom 负值减少了其自身供 CSS 渲染页面时读取的高度，让后续元素跟了上来。
5.
元素的实际高度应该是加上 padding-bottom 后的高度，但上面说了，margin-bottom 减少了供 CSS 读取的高度，所以，父元素能以最高的高度通过 overflow 隐藏掉多余的部分
<div class="box"> <div class="child-orange"> <div class="li">左黄</div> <div class="li">左黄</div> <div class="li">左黄</div> </div> <div class="child-green"> <div class="li">右绿</div> <div class="li">右绿</div> <div class="li">右绿</div> <div class="li">右绿</div> </div> </div>
HTML
.child-orange,.child-green { float: left; width: 50%; margin-bottom: -999px; padding-bottom: 999px; } .child-orange { background: orange; } .child-green { background: green; } .box { overflow: hidden; }
